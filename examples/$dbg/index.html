When printing Alpine object in dev console, you'll notice that, when they are not simple primitive types, they are shown as Proxy object. Proxy are special javascript objects used by Alpine to offer reactivity but they can confuse people. Open the dev console and try to click the 2 buttons below to see the difference

        <div x-data="{ name: 'Ryan', user: { name: 'Example' }, list: [{foo: 'foo'}, 'bar'] }">
            <button @click="console.log(name, user, list)">Without $dbg</button>
            <button @click="$dbg(name, user, list)">With $dbg</button>
        </div>
